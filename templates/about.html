{% translate "What makes PleaseFix different" %}

{% translate "Public by default — issues belong to everyone they affect" %}

{% blocktranslate %}An official complaint is a private conversation between one person and one agency. A PleaseFix issue is a shared community record: anyone can see it, add photos and updates, subscribe, or follow up on an issue an agency already "closed". Duplicate reporters become subscribers instead of noise.{% endblocktranslate %}

{% translate "Evidence that can't be quietly closed" %}

{% blocktranslate %}It has happened that a council closed tickets as "done" when the contractor had done nothing. On PleaseFix the trail is public and permanent: dated photos, the date each complaint was lodged and with whom, every status change. The record outlives the agency's ticket — that is the point.{% endblocktranslate %}

{% translate "Cross-jurisdictional by design" %}

{% blocktranslate %}Government systems are one-agency silos; coordinating them has historically taken joint task forces or entire new agencies. One broken streetlight on a federal road near a rail station can genuinely involve three bodies. PleaseFix tracks one issue across every responsible agency at once — each filing with its own official reference number and status.{% endblocktranslate %}

{% translate "Community status ≠ official status" %}

{% blocktranslate %}Each agency filing carries that agency's status (filed, rejected, "selesai"). The issue's status belongs to the community: it closes when the problem is verifiably fixed, not when a ticket is. "Agency says fixed, community says not" is representable — and visible.{% endblocktranslate %}

{% translate "Tags beyond government categories" %}

{% blocktranslate %}Issues carry community tags no agency taxonomy supports — accessibility, CEDAW, public transport, environment — so advocates can see, for example, every accessibility issue in one neighbourhood at once.{% endblocktranslate %}

{% translate "It teaches jurisdiction" %}

{% blocktranslate %}Who is responsible for what is tribal knowledge. PleaseFix explains it — per issue type and per area — and records the official reference numbers, so the government-side trail is trackable from the public one and the next reporter files smarter.{% endblocktranslate %}

{% translate "How it works" %}

1 · {% translate "Report" %} 2 · {% translate "Route" %} 3 · {% translate "Dispatch" %} 4 · {% translate "Track" %} 5 · {% translate "Verify" %}

{% blocktranslate %}Report with a photo and a pin on the map (no account needed — a phone number or email is enough). PleaseFix works out the responsible agencies from the location and category, dispatches to each of them, and records every filing. Updates flow back — by agency reply, by staff liaison, or by neighbours adding evidence — and the community verifies the fix before the issue closes.{% endblocktranslate %}

{% translate "Built to be built on" %}

{% blocktranslate %}PleaseFix is open source (Apache-2.0) and deliberately contributor-friendly: Django + HTMX, strict typing, one docker compose up for the whole stack. The public API is a first-class product — build your own reporting app, neighbourhood dashboard, or advocacy view on the same public data.{% endblocktranslate %}
