PleaseFix

The public, community-owned issue tracker for Malaysia. Open until it's actually fixed.

Penjejak isu komuniti yang terbuka — kekal terbuka sehingga benar-benar selesai.

Official channels close tickets. Communities fix problems.

"I cannot cross the road from the LRT station because of a barrier."

Official channel

  • Filed with one agency
  • "Not our jurisdiction / private company"
  • Ticket closed. Barrier still there.

PleaseFix

  • SISPAA: rejected · MPSJ: filed · Prasarana: filed · Councillor: raised
  • Every filing, date, and photo on one public record
  • Issue stays open until the crossing exists.

What makes it different

  • Public by default. Every issue is a shared community record — anyone can see it, add photos and updates, or follow up on an issue an agency already "closed". A complaint to an agency is a private conversation; a PleaseFix issue belongs to everyone it affects.
  • Evidence that can't be quietly closed. Councils have closed tickets as "done" when the contractor did nothing. A public tracker keeps the dated photos and the dates every complaint was lodged — permanently, visibly.
  • Cross-jurisdictional by design. .gov.my systems are silos; coordinating them takes joint task forces (or, once, a whole agency — SPAD). PleaseFix tracks one issue across every responsible agency at once, with each official reference number attached.
  • Issues can be goals, with dependencies. No government system can represent "I want to walk from A to B":
I can't walk from A to B [main issue — open] ├─ blocked by: no crossing at Y (MPSJ — filed) └─ blocked by: streetlights out at Z (TNB — filed)
  • Community metadata. Tags beyond any agency's category codes — accessibility/a11y, CEDAW, public transport, environment — so advocates can see, e.g., every accessibility issue in Subang at once.
  • It teaches jurisdiction. Who is responsible for what is tribal knowledge. PleaseFix explains it per issue type and area, and records the official reference numbers so the government-side trail is trackable from the public one.

Developers: this is built to be built on

PleaseFix is open source and deliberately contributor-friendly: Django + HTMX, strict types, one docker compose up for the entire stack, and CI that handles formatting and style so reviews are about ideas. Channel adapters and dispatch adapters are designed as safe first contributions — one agency's integration breaking never takes down another's.

Prefer building your own thing? The public API is the product: an OpenAPI-typed /api/v1, generated SDKs, thin client templates, and a hosted sandbox with seeded Malaysian data — build a neighbourhood dashboard or advocacy view without ever installing PostGIS. Community clients get listed; recognition is the currency.
